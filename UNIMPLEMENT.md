## 📌 백로그 개요

이 문서는 StayOn 앱의 **향후 개발 계획**을 기록합니다.  
각 기능은 우선순위, 난이도, 비즈니스 가치로 평가되어 있습니다.

### 우선순위 기준
- **High:** 핵심 기능, 사용자 만족도 높음
- **Medium:** 유용하지만 필수는 아님
- **Low:** 나중에 추가 가능

### 난이도 기준
- ★☆☆☆☆ : 매우 쉬움 (몇 시간)
- ★★☆☆☆ : 쉬움 (1-2일)
- ★★★☆☆ : 중간 (2-5일)
- ★★★★☆ : 어려움 (1-2주)
- ★★★★★ : 매우 어려움 (2주 이상)

## 🎯 미구현 기능 목록

### 🔧 기존 기능 개선 (3개)

#### 1. 데이터 영구 저장 (로컬 데이터베이스)

| 항목 | 내용 |
|------|------|
| **기능명** | 📦 SQLite 데이터베이스 연동 |
| **설명** | 현재 세션 상태로만 저장되는 데이터를 SQLite 데이터베이스에 영구 저장하여 브라우저 새로고침 후에도 데이터 유지 |
| **우선순위** | High |
| **난이도** | ★★★☆☆ |
| **비즈니스 가치** | 🎯 **데이터 안전성 확보** - 사용자 신뢰도 향상, 실제 업무 사용 가능 |
| **예상 작업량** | 3-4일 |
| **기술 스택** | SQLite3, SQLAlchemy (ORM) |

**구현 계획:**
```python
# 데이터 모델 설계
class Todo(Base):
    id = Column(Integer, primary_key=True)
    title = Column(String(100))
    deadline = Column(DateTime)
    completed = Column(Boolean)

# CRUD 함수 작성
def add_todo_db(title, description, deadline, priority)
def get_todos()
def delete_todo(todo_id)
```

#### 2. 카테고리/태그 기능

| 항목 | 내용 |
|------|------|
| **기능명** | 🏷️ 할 일 분류 및 검색 |
| **설명** | 할 일을 카테고리(업무, 개인, 학습)나 태그(#긴급, #팀프로젝트)로 분류하여 관리 효율성 증대 |
| **우선순위** | Medium |
| **난이도** | ★★☆☆☆ |
| **비즈니스 가치** | 📈 **대량 데이터 관리 효율화** - 100개 이상의 할 일도 쉽게 구분 |
| **예상 작업량** | 2-3일 |
| **기술 스택** | 다중선택 체크박스, 다대다 관계 DB |

**추가할 UI 요소:**
```
[카테고리 선택] ▼     회사 / 개인 / 학습
[태그 입력]     _____  #긴급 #회의준비
```

#### 3. 반복 작업 (되풀이) 기능

| 항목 | 내용 |

